# 네이버 플러스 스토어 마케팅 분석 프로젝트

> 이 파일은 Claude Code가 매 대화마다 자동으로 읽음.
> 퍼포먼스 / CRM / 그로스 팀이 광고 집행·분석에 활용.

---

## 1. 프로젝트 개요

**조직**: 네이버 플러스 스토어 마케팅실
**사용 팀**: 퍼포먼스 / CRM / 그로스
**분석 대상**: 외부 유료 매체 + 네이버 자체 매체 + Braze(CRM) 통합 퍼널

### 분석 목적
- 주간 채널 효율 리포트 (ROAS / CAC / LTV)
- 플러스 멤버십 신규 가입 유도 캠페인 최적화
- 퍼포먼스 → CRM 연계 리텐션·재구매 분석
- 소재·크리에이티브 A/B 테스트 성과 판정
- 월간 예산 재분배 의사결정 근거

### 북극성 지표 (North Star)
**활성 플러스 멤버 MAU × 월간 구매액 (GMV)**
- 퍼포먼스: 신규 플러스 가입 + 첫 구매 전환
- CRM: 구독 유지율, 재구매율
- 그로스: 온보딩 퍼널 이탈 최소화

## 3. 네이밍 컨벤션 ⭐

### 3-1. 채널 코드 매핑

| 채널명(내부) | AppsFlyer 미디어소스 | 약어 | 브랜드컬러 | 분류 |
|---|---|---|---|---|
| 구글 | `googleadwords_int` | GGL | `#4285F4` | 외부 |
| 메타 | `Facebook Ads` | META | `#1877F2` | 외부 |
| 네이버 | `naver_search` | NVR | `#03C75A` | **자체 매체** |
| 카카오(예정) | `kakao_moment` | KKO | `#FEE500` | 외부 |
| 틱톡(예정) | `TikTok_Ads` | TTK | `#000000` | 외부 |

> **⚠️ 자체 매체는 별도 트래킹**
> 네이버 GFA·검색광고는 그룹사 매체라 "외부 채널 효율 비교"에서 보통 제외.
> 전체 집계 시엔 포함하되, 채널 랭킹·ROAS 비교에는 분리 표기.

### 3-2. 캠페인 ID 포맷

```
{채널약어}_CMP_{번호}_{목적}
예: GGL_CMP_01_플러스가입, META_CMP_02_리타겟팅
```

**목적 값** (네이버 플러스 스토어 기준):
- `플러스가입` — 플러스 멤버십 신규 가입 유도 (퍼마 핵심)
- `첫구매` — 가입 후 첫 구매 전환
- `재구매` — 기존 멤버 리타겟팅
- `리타겟팅` — 앱 방문자 재유인
- `신규유저` — 앱설치 위주
- `룩얼라이크` — 유사 타겟 확장
- `브랜드KW` — 네이버 브랜드 키워드
- `일반KW` — 네이버 일반 키워드

### 3-3. 소재 네이밍 포맷

```
{소재타입}_{카테고리}_{시즌}_{AB}_{버전}
예: VID_플러스멤버십_겨울_A_v2
    IMG_적립혜택_상시_v1  (AB 없음)
```

| 필드 | 값 |
|---|---|
| **소재타입** | `IMG` / `VID` / `CRS`(캐러셀) / `GIF` / `TXT` |
| **카테고리** | `플러스멤버십` / `적립혜택` / `배송혜택` / `할인쿠폰` / `신상품` / `특가` / `리뷰` |
| **시즌** | `상시` / `봄` / `여름` / `가을` / `겨울` / `블프` / `연말` / `신학기` |
| **AB** | `A` / `B` / 생략 (A/B 미적용) |
| **버전** | `v1`, `v2`… (소재 수정 반복 시) |

**활용**: 소재명에서 `_`로 split 해서 5개 속성 컬럼 파생 → 속성별 성과 분석.

### 3-4. 그룹(광고그룹) 네이밍

| 그룹명 | 타겟 정의 |
|---|---|
| `논타겟` | 전국, 20~49세, 관심사 미지정 |
| `유사타겟` | 플러스 멤버 시드 기반 Lookalike 1~3% |
| `리마케팅` | 최근 30일 앱 방문자 중 미가입 |
| `윈백` | 멤버십 해지 후 30~90일 |
| `VIP` | 상위 20% 구매자 (멤버십 유지용) |

### 3-5. Braze 캠페인 네이밍 (CRM)

```
{채널}_{목적}_{세그먼트}_{버전}
예: PUSH_웰컴_신규7일_v1
    KAKAO_윈백_이탈30일_v2
    INAPP_재구매_VIP_v1
```

| 필드 | 값 |
|---|---|
| 채널 | `PUSH` / `KAKAO` / `INAPP`(인앱메시지) / `EMAIL` / `SMS` |
| 목적 | `웰컴` / `온보딩` / `재구매` / `윈백` / `쿠폰지급` / `알림` |
| 세그먼트 | `신규7일` / `활성` / `휴면30일` / `이탈30일` / `VIP` 등 |

## 4. 조인 규칙

### 4-1. 채널 × AppsFlyer (5-key JOIN)

```sql
ON channel.일 = af.일
AND channel.채널 → channel_map → af.미디어소스
AND channel.캠페인 = af.캠페인
AND channel.그룹   = af.그룹
AND channel.소재   = af.소재
```

### 4-2. AppsFlyer × Braze (3-way)

- 키: `appsflyer_id` ↔ Braze `external_id`
- 기간: 캠페인 발송일 기준 **7일 내 전환 이벤트**를 해당 캠페인 기여로 간주
- 중복 귀속: 퍼마·CRM 둘 다 기여한 전환은 **last-touch** 기준 (가장 최근 터치 채널)

### 4-3. 어트리뷰션 윈도우 (업계 표준 + 이커머스)

| 유형 | 윈도우 |
|---|---|
| 클릭 어트리뷰션 | **7일** |
| 뷰 어트리뷰션 | **1일** |
| 플러스 멤버십 가입 ROAS 측정 | **D7** 기본, **D30** 보조 |
| 구매 ROAS 측정 | **D7, D30** 병행 |
| CRM 캠페인 기여 | 발송 후 **7일** |

## 5. 지표 정의

### 5-1. 기본 지표

| 지표 | 공식 | 기준 |
|---|---|---|
| CTR | 클릭 / 노출 × 100 | 채널 보고 |
| CPC | 비용 / 클릭 | 채널 보고 |
| CVR | 전환 / 클릭 × 100 | 전환 정의별로 분리 |
| **CAC** | **비용 / AF 가입** | **AF 기준** (채널 보고 X) |
| **ROAS (D7)** | **AF D7 구매매출 / 비용** | **AF 기준** |
| **ROAS (D30)** | **AF D30 구매매출 / 비용** | **AF 기준** |
| AOV | 매출 / 구매 건수 | - |
| 재구매율 | 2회+ 구매자 / 전체 구매자 | 30일 윈도우 |
| 어트리뷰션 커버리지 | AF 가입 / 채널 가입 × 100 | 70~90% 정상 |

### 5-2. 네이버 플러스 스토어 특화 지표

| 지표 | 정의 |
|---|---|
| **플러스 가입 CPA** | 비용 / 플러스 멤버십 신규 가입 |
| **첫구매 전환율** | 가입 후 7일 내 첫구매 / 신규 가입 |
| **LTV (D90)** | 가입 후 90일 누적 구매액 |
| **LTV:CAC 비율** | LTV(D90) / CAC — **3:1 이상 유지 목표** |
| **구독 유지율** | 3개월차 활성 멤버 / 가입 시점 멤버 |

### 5-3. CRM 전용 지표 (Braze)

| 지표 | 정의 |
|---|---|
| 푸시 허용률 | 옵트인 유저 / 전체 유저 |
| 발송 도달률 | 도달 / 발송 (푸시 토큰 유효성) |
| 개봉률 | 개봉 / 도달 |
| 캠페인 CTR | 클릭 / 도달 |
| CRM 기여 매출 | 발송 후 7일 내 구매매출 (발송그룹) - (컨트롤그룹) |

### 5-4. 효율 임계값 (캠페인 상태 판정)

| 상태 | 조건 | 조치 |
|---|---|---|
| 🟢 **양호** | ROAS ≥ **4.0** (이커머스 기준) | 예산 증액 검토 |
| 🟡 **관찰** | ROAS 2.0 ~ 4.0 | 소재·타겟 튜닝 |
| 🔴 **개선 필요** | ROAS < **2.0** | 일시중단·소재 리프레시 |

> **참고**: 플러스 멤버십 가입 캠페인은 초기 ROAS가 낮아도 LTV 회수 가능 → ROAS(D7) 대신 **LTV:CAC 3:1** 우선 검토.

## 6. 분석 관행 / 비즈니스 룰

### 6-1. 비교 시 분리 원칙

- **자체 매체(네이버 GFA·검색광고)는 외부 비교에서 분리** — 그룹사 매체라 CPC 단가·어트리뷰션 특성 다름
- **브랜드 키워드(`NVR_CMP_01_브랜드KW`)는 ROAS 산정에서 제외** — 검색 의도 강해 ROAS 부풀려짐 (사실상 오가닉에 가까움)
- **네이버 로그인 연동 유입은 MMP 기여가 일부 누락** 가능 → 자체 로그 보조 확인

### 6-2. 시즈널리티 · 이벤트 기간

| 기간 | 특성 | 처리 |
|---|---|---|
| 월초 1~3일 | 프로모션 부스트 | 베이스라인 계산에서 제외 |
| 월급일 (10·15·25일) | 구매 피크 | 별도 스파이크 분석 |
| 주말 | 트래픽·전환율 ↑ (이커머스 특성) | 평일/주말 분리 리포트 |
| 블프·11월 빅세일 | 대규모 이벤트 | 별도 스테이징, 연평균 제외 |
| 연말·명절 | 선물 수요 | 카테고리별 시즈널리티 반영 |

### 6-3. 신규 vs 기존 분리

- 퍼마는 **신규 유저 중심** (앱설치·플러스가입)
- CRM은 **기존 유저 중심** (재구매·리텐션)
- 한 캠페인이 둘 다 타겟팅할 경우 **세그먼트 필터**로 분리 집계

### 6-4. 소재 A/B 테스트 판정 기준

- 최소 샘플: 그룹당 **노출 10만 이상**
- 통계적 유의성: p-value < 0.05 (CTR·CVR 비교 시)
- 최소 운영 기간: **7일** (주중/주말 사이클 포함)

## 7. 알려진 데이터 이슈

### 7-1. 플랫폼별 공통 주의사항

- **AppsFlyer 환불 이벤트** (`negative revenue`) → 분석 전 filter 또는 `revenue > 0` 조건 명시
- **Google Ads 원본 타임존 UTC** → KST 변환 필수 (9시간 오프셋)
- **Meta는 view-through attribution 기본 On** → AF와 숫자 차이 날 수 있음
- **네이버 검색광고는 세션 기반 리포트** → 앱 클릭 1:1 매핑 안 됨

### 7-2. 어트리뷰션 갭 발생 원인

| 갭 원인 | 대처 |
|---|---|
| iOS SKAdNetwork 제약 (iOS 14.5+) | iOS는 MMP 갭 15~25% 일반적 — 별도 캘리브레이션 |
| View-through 집계 차이 | 뷰 1일 윈도우 통일 |
| 네이버 로그인 경유 전환 | 자체 로그 보조 확인 |
| 오가닉 혼입 | 브랜드KW 제외 |

### 7-3. 데이터 품질 체크리스트

매 분석 시작 전 자동 체크:
- [ ] 결측치: 비용/노출 = 0인 행 → 제외 또는 표기
- [ ] 중복: (일, 채널, 캠페인, 그룹, 소재) 키 중복
- [ ] 음수: 환불 제외 매출에 음수 없는지
- [ ] 이상치: 일 비용이 평균 × 3σ 넘는 날

## 8. 도구 · 워크플로우 선호

### 기술 스택
- **조인 · 집계**: **DuckDB** (SQL, CSV/Parquet 직접 쿼리)
- **탐색 EDA**: **pandas** (Jupyter 스타일)
- **시각화**: **plotly** (인터랙티브), matplotlib (정적 PNG)
- **대시보드**: **Streamlit** (로컬, 팀 공유는 localhost 또는 Snowflake 배포)

### 출력 규칙
- **내부 중간 저장**: Parquet (`*.parquet`)
- **공유용 export**: CSV with `utf-8-sig` (엑셀 한글 안 깨지게)
- **그래프**: `outputs/` 폴더에 PNG·HTML
- **색상**: §3-1 브랜드컬러 우선, 중립은 colorblind-safe 팔레트

### 분석 스타일
- **숫자 먼저, 해석 뒤에**
- 작은 탐색은 텍스트·표로만, 대시보드는 **명시적 요청 시에만**
- 금액 포맷: `₩1.2억` / `₩3,450만` (한국식 축약)
- 퍼센트: 소수점 1자리 (`12.5%`)
- ROAS: 소수점 2자리 (`4.52`)
